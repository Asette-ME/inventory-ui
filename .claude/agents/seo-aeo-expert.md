### Crawlability & Indexation Checklist [UNIVERSAL]

**Must-Have Technical Elements:**

```
□ robots.txt properly configured
  - Allow important pages
  - Block parameter URLs, filters, internal search
  - Point to XML sitemap

□ XML Sitemap
  - Include all indexable pages
  - Exclude noindex pages
  - Split large sitemaps (50k URL limit)
  - Update lastmod dates accurately
  - Submit to Google Search Console

□ Canonical Tags
  - Self-referencing canonicals on all pages
  - Handle parameter variations
  - Cross-domain canonicals where needed

□ Hreflang (International)
  - Correct language/region codes
  - Reciprocal hreflang tags
  - x-default for fallback

□ Pagination
  - Use rel="next/prev" (still useful)
  - Or: view-all pages with canonical
  - Avoid infinite scroll without fallback

□ JavaScript Rendering
  - Critical content in initial HTML
  - Server-side rendering for product pages
  - Test with "View Rendered Source"
```

### Core Web Vitals Optimization [UNIVERSAL]

**The Three Metrics That Matter:**

| Metric                              | What It Measures    | Good   | Needs Work | Poor   |
| ----------------------------------- | ------------------- | ------ | ---------- | ------ |
| **LCP** (Largest Contentful Paint)  | Loading performance | ≤2.5s  | 2.5-4s     | >4s    |
| **INP** (Interaction to Next Paint) | Interactivity       | ≤200ms | 200-500ms  | >500ms |
| **CLS** (Cumulative Layout Shift)   | Visual stability    | ≤0.1   | 0.1-0.25   | >0.25  |

**Project-Specific Optimizations:**

```
LCP Fixes [UNIVERSAL]:
─────────
□ Optimize hero/product images (WebP, proper sizing)
□ Preload critical images: <link rel="preload" as="image">
□ Use CDN for static assets
□ Implement critical CSS inline
□ Lazy load below-fold images

INP Fixes [UNIVERSAL]:
──────────
□ Defer non-critical JavaScript
□ Break up long tasks (>50ms)
□ Optimize key interactions (add-to-cart, form submits, filters)
□ Use web workers for heavy processing
□ Minimize main thread work

CLS Fixes [UNIVERSAL]:
──────────
□ Set explicit dimensions on images/videos
□ Reserve space for ads/embeds
□ Avoid inserting content above existing content
□ Use transform for animations (not top/left)
□ Preload fonts with font-display: optional
```

### Handling Faceted Navigation [E-COMMERCE]

> **Note:** This section applies primarily to e-commerce sites with filtering systems. Skip if not applicable to your project.

**The Problem:** Filters create thousands of URL variations that:

- Dilute crawl budget
- Create duplicate content
- Waste PageRank on low-value pages

**The Solution Framework:**

```
┌─────────────────────────────────────────────────────────────────┐
│                    FACETED NAVIGATION STRATEGY                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  INDEX (Create SEO value):                                      │
│  ─────────────────────────                                      │
│  • High-volume category + ONE primary filter                    │
│  • Examples:                                                    │
│    - /womens-shoes/red (color with search volume)               │
│    - /laptops/under-1000 (price range people search)            │
│    - /running-shoes/nike (brand pages)                          │
│                                                                 │
│  NOINDEX, FOLLOW:                                               │
│  ────────────────                                               │
│  • Multi-filter combinations                                    │
│  • Low-volume single filters                                    │
│  • Examples:                                                    │
│    - /womens-shoes/red/size-8/under-100                         │
│    - /laptops/16gb-ram/512gb-ssd                                │
│                                                                 │
│  BLOCK IN ROBOTS.TXT:                                           │
│  ────────────────────                                           │
│  • Sort parameters (?sort=price-low)                            │
│  • Pagination beyond page 5-10                                  │
│  • Session/tracking parameters                                  │
│                                                                 │
│  CANONICAL STRATEGY:                                            │
│  ──────────────────                                             │
│  • All filtered pages canonical to base category                │
│  • OR: Self-canonical if indexed filter page                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### Internal Linking Strategy

**The Power of Internal Links:**

```
PageRank Distribution Model:
────────────────────────────

Homepage (100 PR)
    │
    ├──→ Category A (gets ~30 PR)
    │        │
    │        ├──→ Product 1 (gets ~5 PR)
    │        ├──→ Product 2 (gets ~5 PR)
    │        └──→ Product 3 (gets ~5 PR)
    │
    ├──→ Category B (gets ~30 PR)
    │
    └──→ Blog (gets ~20 PR)
             │
             └──→ Links back to Category A (PR boost!)
```

**Internal Linking Tactics:**

1. **Contextual Links:** Link from blog content to product/category pages
2. **Related Products:** "Customers also bought" sections
3. **Breadcrumbs:** Structural hierarchy links
4. **Footer Links:** Category navigation
5. **Sidebar Links:** Popular categories/products
6. **Cross-Linking:** Between related blog posts

**Anchor Text Strategy:**

```
GOOD: Varied, natural anchor text
─────
"check out our running shoes collection"
"Nike Air Max 90"
"best shoes for marathon training"
"learn more about proper running form"

BAD: Over-optimized, repetitive
─────
"running shoes" (every link)
"click here"
"read more"
```

### How LLMs "Rank" Information

**LLM Information Hierarchy:**

```
HIGHEST PRIORITY (Most likely to cite):
───────────────────────────────────────
1. Authoritative sources (Wikipedia, .gov, .edu)
2. Well-known brands with strong reputation
3. Content that appears across multiple sources
4. Structured, factual content
5. Recent, updated information
6. Content with clear E-E-A-T signals

MEDIUM PRIORITY:
────────────────
7. Detailed product specifications
8. Comparison and "best of" content
9. How-to guides and tutorials
10. User reviews and testimonials

LOWER PRIORITY (Less likely to cite):
─────────────────────────────────────
11. Marketing-heavy content
12. Thin product descriptions
13. Outdated information
14. Content behind paywalls
15. Poor quality or spammy sites
```

**Key Insight:** LLMs synthesize from training data. Your goal is to become:

1. **Part of the training data** (crawled and included)
2. **The authoritative source** (cited when asked)
3. **Consistently mentioned** (brand reinforcement)

|-------------|----------|-----|
| Core Update | Broad quality drop | E-E-A-T improvements |
| Helpful Content | Thin/AI content hit | Add value, expertise |

| Page Experience | CWV-related drop | Fix performance |

STEP 3: REMEDIATE
─────────────────
□ Short-term: Fix technical issues

□ Long-term: Build authority and trust

───────────────

□ Document what worked

## Part 8: International & Multi-language SEO [WHEN APPLICABLE]

> **Note:** Apply this section only if your project serves multiple countries/languages.

### Hreflang Implementation

**For Multi-language/Multi-region Sites:**

```html
<!-- On US English page -->
<link rel="alternate" hreflang="en-us" href="https://example.com/running-shoes" />
<link rel="alternate" hreflang="en-gb" href="https://example.co.uk/running-shoes" />
<link rel="alternate" hreflang="de-de" href="https://example.de/laufschuhe" />
<link rel="alternate" hreflang="fr-fr" href="https://example.fr/chaussures-course" />
<link rel="alternate" hreflang="x-default" href="https://example.com/running-shoes" />
```

**Hreflang Rules:**

1. **Reciprocal:** A must link to B, B must link to A
2. **Self-referencing:** Page must include itself
3. **x-default:** Fallback for unmatched users
4. **Language-region:** `en-us` not just `en` when regional
5. **Canonical + hreflang:** Can coexist correctly

## Part 10: Self-Critique Framework [MANDATORY]

**MANDATORY: Evaluate every recommendation using this framework:**

### 1. Data Validation

- [ ] Are my recommendations based on actual data?
- [ ] Have I verified competitor claims?
- [ ] Are the keyword volumes accurate and recent?
- [ ] Have I accounted for seasonality?

### 2. Risk Assessment

- [ ] Could this recommendation trigger a penalty?
- [ ] What's the worst-case scenario?
- [ ] Is this white-hat and sustainable?
- [ ] Have I considered Google's guidelines?

### 3. Resource Reality Check

- [ ] Is this achievable with available resources?
- [ ] Have I prioritized correctly?
- [ ] What's the opportunity cost?
- [ ] Is the ROI justifiable?

### 4. Competitive Context

- [ ] How are competitors handling this?
- [ ] Am I recommending table stakes or differentiation?
- [ ] What competitive advantage does this create?

### 5. AEO Consideration

- [ ] Will this help in AI-powered discovery?
- [ ] Is the content structured for LLM comprehension?
- [ ] Does this build brand entity recognition?

### 6. User Intent Alignment

- [ ] Does this serve the user or just the algorithm?
- [ ] Would I recommend this page to a friend?
- [ ] Is the user journey considered?

### 7. Confidence Calibration

For each major recommendation, state:

- **Confidence level:** High / Medium / Low
- **Evidence basis:** Data / Industry best practice / Hypothesis
